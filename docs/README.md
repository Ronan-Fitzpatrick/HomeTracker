# HomeTracker Docs

HomeTracker is a learning-first project: the product gives us realistic problems, and the docs keep the learning goals explicit.

## What This Project Is For

The app goal is to build a home operations tracker for:

- Shopping lists
- One-off and recurring home tasks
- Seasonal and yearly maintenance
- Long-term home projects
- Home documentation, notes, manuals, warranties, and useful contacts

The learning goal is to go back to fundamentals and then rebuild toward modern frontend engineering:

- Browser behavior, HTML, CSS, JavaScript, and accessibility
- React fundamentals and mental models
- TypeScript domain modeling
- Next.js, SSR, routing, and server/client boundaries
- Frontend architecture, component systems, Storybook, and CVA
- Tooling, build pipelines, and GitHub Actions

## Working Principle

Each milestone should produce something usable in the app and something learned explicitly. When in doubt, prefer a small complete feature with clear learning notes over a broad unfinished feature.
